Każde rozwiązanie zaczyna się od pustej funkcji ..

function multTwo(a: number, b: number): number {
    // your code here
    return 0;
}

... i zestawu assert, który testuje funkcję po uruchomieniu kodu.

assert.equal(multTwo(3, 2), 6);
assert.equal(multTwo(0, 1), 0);

Celem jest wypełnienie pustej funkcji poprawnym rozwiązaniem.

Co to jest funkcja? Brzmi to trochę zagmatwanie.

Czy może Pan wyjaśnić działanie funkcji za pomocą tego konkretnego zadania?

Z wielką przyjemnością. Właśnie dlatego mamy podpowiedzi na CheckiO!

Proszę sobie wyobrazić, że trzeba pomnożyć dwie liczby i wydrukować (wyprowadzić) wynik tak, aby był widoczny w konsoli. Na przykład, 5 i 6.

console.log(5*6);

Wynik:

30

Teraz zamiast 5 i 6 możemy użyć zmiennych. Do czego? Zobaczą Państwo w dalszych przykładach.

let a = 5;
let b = 6;
console.log(a*b);

Wynik:

30

Można zmienić zmienną a, uruchomić skrypt ponownie i zobaczyć inny wynik

let a = 6;
let b = 6;
console.log(a*b);

Wynik:

36

Można też skopiować kod dwukrotnie (bez let w kopii), zmienić zmienne i zobaczyć wynik dwóch mnożeń

let a = 5;
let b = 6;
console.log(a*b);


a = 6;
b = 6;
console.log(a*b);

Wynik:

30
36

Teraz mogę pokazać, jak funkcja może uprościć powyższy kod.

function multTwo(a, b) {

    return a*b;
}

console.log(multTwo(5, 6));
console.log(multTwo(6, 6));

Wynik:

30
36

Jak Państwo widzą, funkcja multTwo zawiera teraz całą logikę, a Państwo zmieniają tylko argumenty, aby zobaczyć różne wyniki.

Super. Dziękuję, ale proszę wyjaśnić strukturę funkcji bardziej szczegółowo, abym mógł swobodnie używać jej w innych misjach CheckiO.

Oczywiście. Proszę jeszcze raz przedstawić funkcję:

function multTwo(a, b) {

    return a*b;
}

Zawsze zaczyna się od function.

Nazwa funkcji multTwo - tej nazwy będzie Pan używał do wywoływania funkcji w przyszłości.

Dwa argumenty (a, b) - są to zmienne, które otrzymają wartości po wywołaniu funkcji swoje wartości, gdy funkcja zostanie wywołana. Na przykład, gdy wywołujemy funkcję multTwo(5, 6) - wewnątrz funkcji, ustawiamy zmienną a na 5 i zmienną b na 6.

Mogą Państwo również zobaczyć definicje funkcji z podpowiedziami typów.

function multTwo(a: number, b: number): number {

    return a*b;
}

Proszę nie dać się zmylić przez : number. Wyjaśnia ona użytkownikowi (osobie czytającej kod) oczekiwany typ argumentów i typ wyniku wywołania funkcji. Ale na razie proszę je zignorować.

Teraz blok funkcji, który wewnątrz { i } i return mówi, jaki powinien być wynik wywołania funkcji return a*b.

W naszym przykładzie console.log(multTwo(5, 6)) - wynik mnożenia dwóch wartości 5 i 6 zostanie zwrócony i użyty jako pierwszy argument funkcji console.log().

Funkcja może mieć więcej niż jeden wiersz kodu. Na przykład użyjmy jeszcze jednej zmiennej c, w której umieścimy wynik mnożenia i zwrócimy wynik.

function multTwo(a, b) {

    let c = a*b;
    
    return c;
}
Ok. Dziękuję, teraz jest to wystarczająco jasne. Ale kiedy mam poprawną funkcję, co powinienem zrobić dalej?

Na górze edytora kodu ma Pan dwa przyciski, "uruchom kod" i "sprawdź rozwiązanie".

Po naciśnięciu pierwszego z nich - uruchomiony zostanie kod z edytora kodu. Gdy użyją Państwo przycisku "sprawdź rozwiązanie" - system sprawdzi Państwa funkcję i wyświetli panel "gratulacje", gdy będzie ona działać zgodnie z oczekiwaniami.

Gdy coś nie działa zgodnie z oczekiwaniami - panel pokaże wywołanie funkcji, dla której zwracana jest nieprawidłowa wartość
